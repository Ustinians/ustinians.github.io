前端工程师的价值是什么？

前端工程师的价值在于他们能够将设计团队提供的视觉设计和用户体验转化为可执行的代码，并且能够确保该代码可靠、高效地运行在不同的浏览器和设备上。除此之外，前端工程师还负责构建优秀的用户界面，使用户能够简单、直观地与Web应用程序或网站进行交互，从而提高用户满意度和忠诚度。与此同时，前端工程师还可以通过不断学习和实践来推动Web前端技术的进步和创新，促进移动端和Web技术的跨界融合，从而驱动整个互联网行业的发展。

什么是中间人攻击？

中间人攻击（Man-In-The-Middle Attack/Attack, MITM Attack/Attack）是指攻击者与通讯双方的通讯过程中进行拦截和篡改，从而使双方认为自己正与对方直接通信，但实际上所有传输数据都经过了攻击者的篡改和窃取。

中间人攻击可能会在通讯过程中通过伪造数字证书等方式获得通讯双方的信任。一旦攻击者实现了中间人攻击，他就可以窃取或篡改双方通信的数据，读取双方机密信息如账号密码、身份证号、银行卡信息等。

中间人攻击的常见情景包括：公共Wi-Fi网络上的登录、HTTP网站传输数据、SMTP、POP、IMAP等协议等。因此，通常建议用户在使用互联网时使用加密连接方式，如HTTPS、VPN等，并保持软件和系统的更新，以减少中间人攻击的风险。

中间人攻击的流程？

  1. 监听：攻击者在通信的中间位置部署监听设备，如Wi-Fi网络嗅探器等，以截获双方的通信数据包。
  2. 伪造：攻击者会伪装成一个可信的通讯双方参与通信，从而使双方相信自己是真正的通信对象。
  3. 拦截：攻击者会拦截双方通信过程中传输的数据包，修改或替换其中的数据内容，并将伪造的数据重新发送给对方，使得双方都不知道数据包已被篡改。
  4. 窃取：攻击者利用拦截到的通信数据包，获取其中的私密信息，如账号密码、银行卡信息等。
  5. 篡改：攻击者还可以篡改数据包中的信息，比如把转账金额改大、修改登录密码，从而掌控用户的账号和密码。

中间人攻击会涉及到公钥吗？

中间人攻击有可能涉及到公钥。在加密通信中，公钥加密方式通常使用服务器公钥加密来保护通信的安全性。中间人攻击者可以替换服务器公钥发送给客户端，使得客户端的数据其实是被攻击者拦截和篡改过的。因此，公钥的安全性是中间人攻击的一个重要因素。为了减少中间人攻击的风险，可采用一些防御措施，比如使用HTTPS协议、数字证书等。

SSLSniff攻击：

攻击者在网关截获SSL会话，替换服务器公钥证书，将公钥PKey换成自己的公钥PKey，欺骗客户端。客户端使用PKey加密信息并发送会话，中间人用私钥Skey解密客户端返回会话，从而劫持会话。同时，中间人用PKey加密明文会话并返回服务器。

过程如下：

  1. Attacker截获了客户端的say hello，可以把publicKey_attacker返回给客户端，取得客户端的信任，至此Attacker与客户端建立了安全连接。
  2. Attacker冒充客户端向服务器发送say hello，至此Attacker与服务器建立了安全连接。

这种攻击会存在一个问题会被感知到，就是Attacker的证书是伪造的不受信任的证书，所以客户端可以确认是否需要真的连接该服务器。

SSLStrip攻击：

这种攻击相对于攻击一复杂一点，但是也更加厉害，几乎可以在客户端无感知的情况下实施攻击，并且不需要伪造证书；简单来说就是这样：Attacker在客户端与服务器建立连接时，在Attacker与服务器之间形成HTTPS连接，而在客户端与Attacker之间形成HTTP连接，即将SSL层从原HTTPS连接中“剥离”。这样，既避免了在客户端验证证书时难以避免的弹框问题，又能够劫持HTTP明文数据，并同时保证客户端HTTP数据的传输，达到欺骗服务器与客户端的效果。

过程如下：

用户在浏览器地址栏中输入网址时，多数会采用直接输入网址的方式，而忽略了传输所采用的协议。例如，在登录gmail过程中，大多数用户会直接在地址栏中输入www.gmail.com，向Google服务器发送一个HTTP连接请求，而不是输入https：//www.gmail.com， 向服务器发送一个HTTPS连接请求。因此，用户通常接触到HTTPS的方式有两种：一种是 Web上的连接，比如当用户在gmail上输入用户名和密码后，点击的登录键，将用户的用户名和密码以HTTPS的形式POST到服务器。另一种是通过HTTP的302状态。 当客户端向gmail提出HTTP连接请求时，gmail服务器会返回一个REDIRECT网址，https：//www.google.com/accounts/ServiceLogin?service=mail...，用户端在接收到这个URL后，将页面重定位到该网页，并请求HTTPS连接。 从另外一个角度讲，用户通常是通过HTTP向服务器发起HTTPS连接的。而HTTP本身是以明文的形式对外传送，并不能保证数据的安全。因此，可以考虑通过对HTTP进行劫持，来实现对HTTPS劫持的目的。

  1. 客户端向服务器发起HTTP连接请求；
  2. 中间人MITM监听客户端与服务器的HTTP数据；
  3. 服务器返回给客户端的HTTP数据包被在客户端与服务器之间的中间人截获。中间人解析原HTTP数据包，将其中<a href=”https：//...”>替换成<a href=”http//...”>，将 Location： https：//... 替换成Location：http://..，同时记录下所修改的URL，并保存；
  4. 中间人将修改后的HTTP数据发送给客户端；
  5. 客户端向服务器发起HTTP连接请求；
  6. 中间人计算机解析客户端的HTTP连接请求，并与保存文件相比较。当发现存在有已修改过的HTTP URL时，将其替换成原HTTPS URL，并发送给服务器；
  7. 与服务器保持HTTPS连接，回到步骤3；
  8. 与客户端保持HTTP连接，回到步骤4。

中间人攻击如果篡改信息，服务器会发现吗？

如果中间人攻击者成功地篡改了通信传输的信息，服务器端通常无法直接发现这种攻击。因为服务器和攻击者之间的通信是直接的，中间人攻击者的篡改并不会被服务器端所感知。但是，服务器有可能能够通过异常检测、加强的日志分析等方式来发现一些不寻常的行为，如明显的数据篡改或存在异常的数据包传输。此外，服务器也可以启用一些数据完整性和消息认证等机制来检测数据是否被篡改，以降低中间人攻击的风险。

TCP/IP模型与OSI模型的区别与联系？

区别

  1. 层数：OSI模型将网络通信分为7层，而TCP/IP模型将网络通信分为5层。即OSI模型比TCP/IP模型多出了表示层和会话层两层。这也是OSI模型相对于TCP/IP模型而言显得较为冗余的原因之一。
  2. 技术实现的不同：OSI模型更注重于理论和学术领域的规范性，而TCP/IP模型则更加实际和直接，更注重提出实现的协议和解决问题的方法。
  3. 应用场景的不同：OSI模型更多地应用于大型企业网络和公共网络的标准化管理，而TCP/IP模型则更多地应用于广泛的计算机网络，如互联网、局域网和广域网等。

联系

  1. 都是层次结构，分层的目的都是为了使协议的设计和实现更加清晰、模块化。
  2. TCP/IP模型中的各层和OSI模型中的各层都包含特定的协议和标准，而这些协议和标准都是为了实现网络通信而设计的。
  3. 虽然OSI模型和TCP/IP模型的分层结构不同，并且各自都有自己独特的特性，但它们都是网络通信领域的重要参考模型，为网络通信的标准化和规范化做出了重要贡献。

CSS有哪些布局方式？

  1. 盒子模型布局（box model layout）：指定元素的宽度、高度、内边距和边框等属性，通过对盒子模型进行设置来实现布局。
  2. 浮动布局（float layout）：元素通过使用浮动属性，可以脱离标准布局流，浮动到页面的左侧或右侧，同时其他元素会环绕在其周围。
  3. 弹性布局（flexbox layout）：定义一个容器，容器里的项目按照一定的比例分配空间，可以实现自适应的响应式布局。
  4. 网格布局（grid layout）：将页面分成一个个容器来布局，通过定义行和列来划分布局区域，可以实现复杂的布局。
  5. 定位布局（position layout）：使用position属性，指定元素的位置，往往和浮动属性一起使用。
  6. 表格布局（table layout）：将元素当成表格和单元格来布局，可以实现类似传统网页设计的布局效果。
  7. 多列布局（multi-column layout）：将内容分成多个列来显示，可以实现报纸和杂志的页面设计效果。

CSS有哪些默认值？

  1. 字体相关的默认值：
    • font-weight：normal
    • font-size：medium（通常是16px）
    • font-family：根据操作系统和浏览器自定义，如对于 Windows 系统的浏览器，通常为“Segoe UI”，对于 Apple 系统的浏览器，通常为“Helvetica”
    • line-height：normal（通常是字体大小的1.2倍）
  2. 盒模型相关的默认值：
    • margin：0
    • padding：0
    • border：无
  3. 颜色相关的默认值：
    • color：通常为黑色（#000）
    • background-color： 通常为透明（transparent）
  4. 布局相关的默认值：
    • display：通常为 block，但是对于 a、input，button等元素是 inline 或 inline-block
    • position：通常为 static
    • float：无
    • clear：无
    • visibility：visible
    • overflow：visible

需要注意的是，这些默认值是根据浏览器的默认样式表来定义的，而不是由 CSS 规范来规定的。因此，不同的浏览器可能会有不同的默认值。为了确保实现一致的样式效果，建议为自己的 CSS 文件中的样式属性设置初始值。

归并排序的原理是什么?

归并排序（Merge Sort）是一种基于分治思想的排序算法。其基本原理是将一个未排序的列表（数组）递归地分成两个子问题，直到问题规模足够小，然后将这些子问题的解进行合并以获得原始问题的解。

归并排序的实现步骤如下：

  1. 分：将原始问题（未排序的列表）递归地分为两个子问题，直到问题规模足够小，例如列表中只有一个元素。
  2. 治：将两个子问题的解进行合并以获得原始问题的解。

具体的合并过程如下：

  • 建立一个辅助数组，用来存储合并后的结果。
  • 将要合并的两个数组按顺序依次比较，将较小（或较大）的元素放入辅助数组中。
  • 如果其中一个数组中所有元素都已经被合并入辅助数组中，则将另一个数组中剩余的元素依次放入辅助数组。
  • 最后，将辅助数组中的元素复制回原始数组。

归并排序的时间复杂度为 O(n log n)，其中 n 表示列表中元素的个数。它的空间复杂度为 O(n)。可以看出，归并排序的递归深度是 log n，每次递归需要一个长度为 n 的辅助数组，因此总的空间复杂度为 O(n log n)。

快速排序的原理是什么？

快速排序（Quick Sort）也是一种基于分治思想的排序算法。其基本原理是选择一个元素作为基准值（通常选择第一个元素），将列表中所有小于基准值的元素移到基准值的左边，所有大于基准值的元素移到基准值的右边，然后递归地对基准值左右两边的子列表进行排序。

具体的实现步骤如下：

  1. 选择基准值：从列表中选择一个元素作为基准值（通常选择第一个元素）。
  2. 分区：将列表中小于基准值的元素放到基准值的左边，将大于基准值的元素放到基准值的右边，相等的元素可以放到任一侧（通常使用双指针法进行分区操作）。
  3. 递归排序：对基准值左右两边的子列表进行递归排序，直到子列表长度小于等于 1。

快速排序的时间复杂度为 O(n log n)，其中 n 表示列表中元素的个数。最坏情况下的时间复杂度为 O(n^2)，发生在列表已经有序，或者基准值选取不合理的情况下。它的空间复杂度为 O(log n)，因为递归调用的深度是 log n。快速排序是一种原地排序算法，因此它的空间复杂度是比较低的。

为什么快速排序要取一个随机数作为基准数？

快速排序算法的性能与基准值的选择有关。如果每次选择的基准值都是列表中的最小值或最大值，那么每次分割出的两个子列表的长度分别为 n-1 和 1，此时快速排序的时间复杂度就会退化为 O(n^2)，最坏情况发生的概率比较高。而对于随机选择基准值，可以尽可能地避免这种最坏情况的发生，从而保证了算法的性能。

随机选择基准值的优点在于可以减少最坏情况的概率。随机选择基准值可以让算法的平均时间复杂度达到 O(n log n)，可以在很大程度上提高算法的效率。此外，随机选择基准值也让攻击者很难找到攻击的空间，避免了一些安全漏洞。

快速排序再最差的情况下和冒泡排序相比谁更快？为什么？

在最坏的情况下，快速排序的时间复杂度是 O(n^2)，此时快速排序会退化成冒泡排序。但是，快速排序的交换次数要比冒泡排序少得多，因为每次交换只会交换基准值和另一个元素的位置。而冒泡排序在每一轮比较中都有可能进行元素交换，因此在最坏情况下，**冒泡排序的时间复杂度和交换次数都比快速排序更差，均为 O(n^2)**。

因此，尽管快速排序在最坏情况下的时间复杂度和冒泡排序相同，但是快速排序的交换次数要比冒泡排序少得多，相对来说更快一些。此外，在平均情况下，快速排序的时间复杂度要比冒泡排序更小，因此快速排序是一种更加高效的排序算法。

时间复杂度是怎么计算的？

计算算法的时间复杂度通常是通过分析算法中的基本操作次数来完成的，这些基本操作通常是循环、条件分支、加减乘除等简单的指令。我们针对输入规模 n，通过分析算法中所有基本操作执行的次数，得到一个关于 n 的函数 T(n)，就称之为算法的时间复杂度。

在计算时间复杂度时，通常只关注最高阶项，因为它反映了算法的增长速度。比如，对于 T(n) = 5n^2 + 3n + 2，我们只关心 n^2 这一项，而忽略其他项，因为当 n 足够大时，其他项相对于 n^2 来说会变得微不足道。

常见的时间复杂度有：

  • 常数时间复杂度 O(1)
  • 对数时间复杂度 O(logn)
  • 线性时间复杂度 O(n)
  • 线性对数时间复杂度 O(nlogn)
  • 平方时间复杂度 O(n^2)
  • 指数时间复杂度 O(2^n)
